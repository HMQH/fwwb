用户上传文件存储目录说明（多模态附录）

————————————————————————————————————————————
一、仓库中的固定目录

仓库根目录下已创建：

  storage/
    uploads/

说明：
- 具体子目录由服务端在每次提交时创建；`storage/uploads/.gitkeep` 用于 Git 跟踪空根目录。
- 实际用户文件由仓库根 `.gitignore` 忽略，不提交。

————————————————————————————————————————————
二、运行时路径约定（每次提交一批次）

根路径：`<项目根>/storage/uploads/`（或环境变量 `UPLOAD_ROOT`）。

目录层级（先用户、再「本次提交」批次目录，再按类型分子目录）：

  uploads/
    {user_id}/                    # UUID，与 public.users.id 一致
      {batch_id}/                 # 本次提交独占目录：UTC 精确到秒 YYYYMMDDHHmmss；同秒冲突时为 YYYYMMDDHHmmss_1、_2…
        text/                     # 附录 · 文本类文件（可多文件）
          {uuid}.{ext}
        audio/                    # 附录 · 音频
          {uuid}.{ext}
        visual/                   # 附录 · 图片（与历史命名一致）
          {uuid}.{ext}
        video/                    # 附录 · 视频
          {uuid}.{ext}

不再使用「先 user_id/audio/yyyyMM」这类按月卷；批次目录由服务端在写入前分配。

————————————————————————————————————————————
三、数据库字段表 `detection_submissions`：

- `storage_batch_id`：与磁盘上 `{batch_id}` 一致，便于对照。
- `text_paths` / `audio_paths` / `image_paths` / `video_paths`：jsonb 字符串数组，每项为相对 `uploads/` 的路径。

正文粘贴内容在 `text_content`（可空）；语音转写不在本提交接口中采集。

————————————————————————————————————————————
四、权限与安全（实现时）

- 仅服务端在校验登录用户后，在其本人 `user_id` 下创建目录并写入；禁止客户端指定任意路径。
- 下载/读取接口需校验资源属于当前用户。
