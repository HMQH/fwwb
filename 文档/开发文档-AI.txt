基于《A30-多模态反诈.pdf》的功能与页面规划（AI整理）

一、结论
当前项目如果只停留在“登录/注册 + 文本/图片分析 Demo”，还不足以完整覆盖赛题要求。
根据赛题原文，产品必须尽量形成“感知 → 决策 → 干预 → 进化”的完整闭环，因此建议补齐以下五大能力：
1. 多模态输入至少三种：文本、图片/截图、音频；
2. 统一的风险识别与分级预警页面；
3. 监护人联动与高风险干预链路；
4. 安全监测报告、历史记录、通知中心；
5. 案例库/知识库更新与评测管理能力。

二、从赛题中提炼出的“硬性需求”
1. 支持至少三种模态输入：
   - 文本：聊天记录、短信、对话内容；
   - 音频：实时通话、语音消息、录音文件；
   - 视觉：图片、屏幕截图、视频帧或视频流。
2. 要有“智能识别与决策引擎”：
   - 意图识别；
   - 知识库检索；
   - 多模态融合判别；
   - 个性化风险评估。
3. 要有“实时干预与交互”：
   - 分级预警；
   - 监护人联动；
   - 安全监测报告。
4. 要有“自适应进化模块”：
   - 新诈骗案例导入；
   - 知识库更新；
   - 持续评测与优化。
5. 要能支撑比赛演示视频中的完整链路：
   “诱导内容输入 → 风险识别 → 预警 → 监护人通知 → 报告生成”。

三、建议新增的产品功能（按优先级划分）

（一）P0：必须有，直接决定是否满足赛题
1. 登录/注册/身份建档
   - 手机号注册登录；
   - 生日推导角色（儿童/青壮年/老人）；
   - 基础资料完善：昵称、性别、职业/身份标签（学生、财会、老人等可选）；
   - 首次登录引导。

2. 多模态风险分析中心
   - 文本分析：粘贴聊天记录、短信、网页内容；
   - 图片分析：上传聊天截图、转账截图、宣传海报、二维码页面；
   - 音频分析：上传语音消息、录音，或录制一段音频后提交；
   - 统一提交入口：用户从首页进入分析中心，选择模态后完成检测。

3. 风险识别结果页
   - 风险等级：低 / 中 / 高；
   - 诈骗类型识别：投资理财、刷单返利、虚假贷款、冒充客服、AI 合成语音、游戏交易等；
   - 风险证据解释：命中的关键词、可疑话术、图像特征、音频特征；
   - 防护建议：不要转账、不要点链接、联系官方、保留证据等；
   - 一键操作：再次检测、保存记录、通知监护人、导出报告。

4. 分级预警与干预
   - 低风险：给出提醒和学习建议；
   - 中风险：醒目提示并建议停止操作；
   - 高风险：强提醒 + 监护人联动入口 + 报警/客服/紧急联系人快捷按钮；
   - 保留“干预结果”字段：用户是否采纳建议、是否联系监护人。

5. 监护人联动
   - 添加监护人信息：姓名、关系、手机号；
   - 风险事件通知：高风险时向监护人发送应用内消息/短信占位/演示通知；
   - 监护人事件详情：看到风险摘要、时间、建议动作；
   - 一键联系当事人。

6. 安全监测报告
   - 单次报告：对某次分析生成详细风险报告；
   - 周期报告：最近一周/最近一月风险统计；
   - 报告内容：风险等级、诈骗类型、识别依据、建议、时间线、处理结果；
   - 支持导出/分享（比赛展示很有用）。

7. 历史记录与通知中心
   - 历史检测记录列表；
   - 风险事件详情页；
   - 通知中心：风险提醒、监护人联动消息、系统更新消息；
   - 支持按时间、风险等级、诈骗类型筛选。

8. 权限与设备能力管理
   - 麦克风权限；
   - 相册/相机权限；
   - 通知权限；
   - 如果后续做视频流/实时监听，还要预留录音与悬浮提醒权限说明页。

（二）P1：强烈建议有，能显著提升完整度与比赛展示效果
1. 角色化防护配置
   - 面向老人、儿童、青壮年提供不同提醒样式与提示文案；
   - 针对学生、财会、独居老人等身份提供差异化风险策略。

2. 反诈知识库/案例库页面
   - 展示典型诈骗剧本；
   - 提供案例解读、法律法规摘要、防骗建议；
   - 支持“相似案例推荐”，与识别结果页联动。

3. 反诈演练/案例模拟页
   - 用于比赛演示：模拟收到诈骗短信、诈骗语音、诈骗截图；
   - 一键导入测试样本，触发完整识别链路；
   - 也可作为用户教育页面。

4. 用户反馈闭环
   - “识别结果是否准确”反馈；
   - 用户可标记误报/漏报；
   - 后台可回收这些反馈用于模型与知识库优化。

5. 智能问答助手
   - 用户可直接提问：“这个电话像诈骗吗？”“遇到贷款客服要怎么办？”；
   - 与知识库检索配合，提供更像助手的交互体验。

（三）P2：后台/评测/进化模块，强加分项
1. 案例库管理
   - 上传新的文本、图片、音频案例；
   - 标注诈骗类型、风险标签、黑白样本属性；
   - 导入向量数据库。

2. 模型与提示词配置
   - 管理系统 Prompt；
   - 调整阈值；
   - 查看不同模态识别策略。

3. 评测中心
   - 使用自建测试集批量跑评测；
   - 输出准确率、F1、响应时间、误报率；
   - 满足比赛材料中的评估报告要求。

4. 风险事件后台
   - 查看近期用户风险事件；
   - 统计诈骗类型分布；
   - 统计高风险用户、通知次数、误报反馈。

四、建议设计的页面（移动端）

（一）基础账户与初始化页面
1. 启动页 / 品牌页
   - 展示产品名称、定位；
   - 检查登录态。
2. 登录页
3. 注册页
4. 首次建档页
   - 完善昵称、生日、角色标签、是否需要监护人守护。
5. 权限授权页
   - 麦克风、相册、通知。

（二）核心业务页面
6. 首页 / 风险分析中心
   - 三个主要入口：文本检测、图片检测、音频检测；
   - 最近风险摘要；
   - 快捷入口：历史记录、知识库、监护人、报告。
7. 文本检测页
   - 粘贴文本、选择来源（短信/聊天/网页）；
   - 一键分析。
8. 图片检测页
   - 上传截图/拍照；
   - 支持展示原图和识别结果。
9. 音频检测页
   - 上传录音文件；
   - 录制语音后提交；
   - 展示转写内容与分析结果。
10. 风险结果页
   - 风险等级、诈骗类型、识别依据、建议动作、相似案例。
11. 高风险干预页
   - 高风险时强化展示；
   - 一键通知监护人；
   - 一键联系官方/报警/家人。
12. 安全报告页（单次报告）
13. 历史记录列表页
14. 历史记录详情页
15. 通知中心页
16. 监护人管理页
   - 添加/编辑/删除监护人；
   - 查看已通知状态。
17. 反诈知识库页
   - 诈骗类型分类；
   - 典型案例；
   - 防骗指南。
18. 演练/模拟页（推荐）
   - 导入测试案例，便于比赛演示完整链路。
19. 个人中心页
   - 个人资料、角色、设置、退出登录。
20. 设置页
   - 通知开关、隐私说明、权限管理、关于我们。

五、建议设计的页面（监护人视角，可与移动端合并）
1. 监护人首页
   - 展示被守护对象风险状态。
2. 风险通知列表页
   - 最近收到的高风险事件。
3. 风险事件详情页
   - 时间、诈骗类型、风险说明、建议动作。
4. 远程干预页
   - 一键拨打电话、一键发送提醒、查看处理进度。

说明：
如果项目时间有限，不一定单独做一套“监护人 App”。也可以在同一个 App 中通过“监护人模式”或“通知详情页”完成展示。

六、建议设计的页面（后台 Web 端，强烈建议至少做简版）
1. 后台登录页
2. 数据总览页
   - 风险事件数、诈骗类型分布、识别准确率、误报率、平均响应时间。
3. 案例库管理页
   - 文本/图片/音频案例上传、分类、标注。
4. 知识库导入页
   - 新案例清洗、导入、向量化状态。
5. Prompt / 模型配置页
6. 评测中心页
   - 测试集运行结果、F1、准确率、响应时间。
7. 风险事件审计页
8. 用户与监护人管理页

七、推荐的核心用户流程（建议作为比赛 Demo 主线）
流程一：普通用户主动检测
注册/登录 → 完善资料 → 首页选择文本/图片/音频 → 提交分析 → 查看风险结果 → 生成报告 → 保存历史。

流程二：高风险联动流程
提交诈骗内容 → 系统判定高风险 → 进入强提醒页 → 一键通知监护人 → 监护人收到消息 → 查看事件详情 → 联系用户干预。

流程三：知识进化与评测流程
后台导入新诈骗案例 → 案例入库/向量化 → 前台识别能力更新 → 测试集批量跑评测 → 输出报告。

八、对当前项目的缺口判断
当前已有：
- 登录/注册；
- 基础角色推导；
- 前端结构雏形；
- 文本/图片方向的 Demo 基础。

当前明显缺少：
- 音频检测页面与接口；
- 统一风险结果页；
- 高风险干预页；
- 监护人管理与通知页；
- 安全报告页；
- 历史记录/通知中心；
- 知识库/案例库页面；
- 后台案例导入与评测页面。

十、最终建议
如果目标是“能参赛、能演示、能讲清楚闭环”，建议产品至少包含：
- 登录/注册；
- 多模态分析（三种模态）；
- 风险结果与解释；
- 高风险预警；
- 监护人联动；
- 安全报告；
- 历史记录；
- 知识库/案例页；
- 简版后台评测或案例管理页。

这样才能比较完整地体现赛题强调的：
“多模态输入 + 智能识别 + 实时干预 + 自适应进化”。
