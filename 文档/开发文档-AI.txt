《反诈守护》移动端 — 开发文档（与当前代码 front/app 对齐）

说明：本文描述仓库内已实现界面与布局约定；赛题全文能力（风险结果页、监护人、报告等）见文末「规划与缺口」。

————————————————————————————————————————————
一、全局导航与底边栏（Tabs）

路由根：`app/_layout.tsx` 为 Stack（无顶栏），子路由含 `(auth)` 与 `(main)`。

登录后进入 `(main)/_layout.tsx`，使用 **底部 Tab 栏**（`expo-router` Tabs），样式要点：
- 背景：浅色 surface，顶部分割线 `palette.line`。
- 高度：约 62px + 底部安全区（刘海/手势条）。
- 选中态：标签与图标为主题色 `accentStrong`，项背景为 `accentSoft`。
- 键盘弹出时可隐藏底栏（`tabBarHideOnKeyboard: true`）。

底栏 **可见三项**（从左到右）：

| 路由名   | 标签文案 | 图标（MaterialCommunityIcons）     | 说明           |
|----------|----------|-------------------------------------|----------------|
| index    | 首页     | home-variant / home-variant-outline | 主入口、摘要   |
| records  | 记录     | history                             | 历史/消息列表（当前为演示数据） |
| profile  | 我的     | account-circle / account-circle-outline | 资料与占位入口 |

**隐藏项**（不出现在底栏，`href: null`）：
- **submit**：「提交检测」全屏，由首页卡片 `router.push("/submit")` 进入；返回用页内「返回」或系统返回。

未登录访问 `(main)` 会重定向到 `/login`。

————————————————————————————————————————————
二、认证区（无底栏）

路径：`app/(auth)/login.tsx`、`register.tsx`。

布局约定（与 `AuthShell` / `AuthBackdrop` 一致）：
- **垂直居中**：主表单区在屏幕中部偏上，避免贴底被键盘遮挡。
- **背景**：淡蓝渐变 + 可选装饰（AuthBackdrop）。
- **品牌区**：顶部标题「反诈守护」/「创建账户」等，副标题说明用途。
- **表单**：手机号、密码等为 `AuthInput` 风格；注册含生日、昵称、协议勾选等。
- **主按钮**：全宽主色按钮，圆角与 `radius` 一致；下方次要「去注册 / 去登录」文字链。
- **登录成功**：写入会话后进入 `/(main)` 首页。

三、首页 `app/(main)/index.tsx`

**页面结构（自上而下，ScrollView 内）：**

1. **顶栏（topBar）**  
   - 左：品牌块 — 圆形 Logo 容器 + 文案「反诈守护」「首页」两行。  
   - 右：「实时防护中」胶囊（绿点 + 文案），与主色协调。

2. **主问候卡片（heroCard）**  
   - 大圆角、主色背景、浅色文字。  
   - 内容：问候语（含昵称）、角色标签（`roleMeta`）、一句角色说明（tone）。

3. **多模态检测入口（单卡片 entryCard）**  
   - 横向：左图标（layers-triple-outline）+ 中间标题/说明/副文案 + 右 chevron。  
   - 整卡可点击 → `router.push("/submit")`。  
   - 说明：一次提交可含文字、图片、音频、视频附录（与后端约定一致）。

4. **区块「最近摘要」**  
   - 小节标题 `sectionTitle`。  
   - **groupCard**：列表行，每行左侧小圆点 + 标题 + 详情；行间分隔线。  
   - 数据按 `user.role`（child / youth / elder）切换文案，为静态演示。

**布局要点：**
- 水平内边距统一 16；区块之间 `gap` 约 18。  
- 卡片使用 `panelShadow` + 细边框，圆角 `radius.lg` / `radius.xl`。  
- 主字体：`fontFamily.display` 标题，`body` 正文。

四、记录页 `app/(main)/records.tsx`

**页面结构：**

1. **页头**：标题「记录」+ 副标题（说明时间线/筛选占位）。  
2. **时间线列表**：每条为横向卡片 — 左侧图标 + 类型标签 + 时间；右侧为标题与详情。  
3. **类型**：文本检测 / 图片检测 / 语音检测 / 系统消息 / 风险消息（不同图标与颜色占位）。  
4. 当前为 **按角色生成的静态 mock**，未接真实后端列表。

**布局要点：**
- 纵向 ScrollView，卡片左对齐图标列 + 右侧弹性内容区。  
- 行距与首页 groupCard 风格统一。

五、我的 `app/(main)/profile.tsx`

**页面结构：**

1. **页头**：「我的」+ 副标题（资料、监护人、报告、设置入口说明）。  
2. **资料大卡（profileCard）**：上排 — 左 Logo 圆形容器 + 右昵称、角色标签、角色描述。  
3. **「资料」** groupCard：多行信息行，每行：图标 + 标签 + 值（昵称、脱敏手机号、生日、守护模式）。  
4. **「守护与工具」**：可点击行列表（监护人管理、安全报告、设置）— 占位文案，未实现真实跳转。  
5. **底部**：退出登录按钮（主色描边或填充），加载态防重复点击。

**布局要点：**
- 信息行：左图标+标签，右对齐数值；行间分隔线。  
- 可点击行：整行 Pressable，右侧 chevron。

六、提交检测 `app/(main)/submit.tsx`（隐藏 Tab，全屏）

**页面结构：**

1. **顶栏**：仅「返回」+ 返回上一页（不出底栏）。  
2. **标题区**：大标题「提交检测材料」+ 副标题（格式说明、两列网格说明）。  
3. **正文卡片**：多行 `TextInput`（粘贴聊天/短信等）。  
4. **附录分块（每块一个 card）**（顺序固定）：
   - 文本类 / 图片（相册）/ 音频 / 视频（相册 + 可选文件管理）。  
   - 每块：小节标题 → 主操作按钮（添加）→ **附录预览网格**（两列 `flexWrap`，缩略图约 48% 宽 + 文件名 + 移除）。  
   - 图片可点缩略图 → **全屏 Modal 大图**（黑底、顶栏关闭 + 文件名）。  
5. **底部主按钮**：「提交到服务端」；提交成功弹窗后清空表单。

**布局要点：**
- 附录 **禁止** 横向长条占满一屏：采用 **2 列网格**，减少纵向高度。  
- 卡片内：`label` → `addBtn` → `previewGrid`；视频区可有 **主按钮 + 次要按钮** 两行。  
- 安全区：顶部 `SafeAreaView edges={['top']}`；大图 Modal 含 `top/bottom` 安全区。

七、设计令牌（与 `shared/theme.ts` 一致）

- **背景**：`palette.background`（浅蓝灰）。  
- **卡片 surface**：白底 + `palette.line` 边框。  
- **主色**：`accent` / `accentStrong` 用于按钮、强调图标。  
- **圆角**：`radius.md`～`radius.xl`；胶囊 pill 用于状态标签。  
- **触控**：列表行、卡片入口保留足够点击区域，附录移除按钮 `hitSlop` 放大。

八、规划与缺口（相对赛题 / 开发文档-AI 旧版）

以下能力尚未在 App 内独立成页，可后续迭代：  
- 统一风险结果页、分级预警、监护人联动、安全报告、通知中心、知识库、后台评测等。  
- 「记录」页对接真实 `detection_submissions` 列表与详情。  
- 「我的」内监护人 / 报告 / 设置跳转真实子页面。

本文档随界面迭代更新；实现以 `front/app` 与 `front/shared/theme.ts` 为准。
